## Контекст проекта

INNER — Telegram-бот для саморефлексии. ИИ-наставник ведёт пользователя через структурированные диалоги («треки»), отвечая голосом выбранного наставника («маска»).

**Стек:** Python, python-telegram-bot, DeepSeek API (совместим с OpenAI SDK), JSON-треки, .env для секретов.

**Структура:**
- `inner_bot.py` — точка входа, логика бота
- `deepseek_api.py` — вызов ИИ (поддерживает MOCK_MODE=true для тестов без ключа)
- `config.py` — переменные окружения
- `utils/session_manager.py` — состояние сессий пользователей
- `tracks/*.json` — сценарии сессий (5 вопросов каждый)
- `masks/*.txt` — системные промпты для голосов наставников
- `.env` — секреты (не в git), `.env.example` — шаблон

### Если нужно понять структуру проекта
Прочитай ключевые файлы: `inner_bot.py`, `config.py`, `utils/session_manager.py`, один трек из `tracks/`, одну маску из `masks/`. Объясни как работает проект целиком — от команды /start до ответа наставника. Используй простые слова, без жаргона.

---

### Если нужно запустить или проверить запуск
Проверь:
1. Существует ли `.env` (не `.env.example`)
2. Установлены ли зависимости (`venv/` существует)
3. Установлен ли `MOCK_MODE` в `.env`

Выдай конкретные команды для запуска с учётом того, что найдено. Если `.env` не заполнен — укажи какие поля обязательны.

Команды запуска:
```bash
source venv/bin/activate
python inner_bot.py
```

Перед правкой прочитай затрагиваемый файл. Вноси изменения точечно. После изменения:

### Если нужно добавить трек
Создай файл `tracks/название_трека.json` по структуре:
```json
[
  {"step": 1, "question": "Вопрос наставника"},
  {"step": 2, "question": "..."},
  ...
]
```
Рекомендуемое количество шагов: 5–7. Вопросы должны вести к осознанию, а не давать готовые ответы. После создания — добавь трек в таблицу README и зарегистрируй в словаре `TRACKS` в `inner_bot.py`.

---

### Если нужно добавить маску наставника
Создай файл `masks/название.txt`. Маска — это системный промпт для ИИ. Укажи: стиль речи, чего избегать, как реагировать на ответы пользователя, длину ответа (1–3 предложения). После создания — добавь маску в `MASKS` в `inner_bot.py` и в таблицу README.

---

### Если нужно обновить README
Прочитай текущий `README.md` и актуальное состояние проекта (треки, маски, стек). Обнови только то, что изменилось: таблицы треков и масок, инструкцию по установке если что-то поменялось. Не переписывай концепцию и миссию проекта.

### Если нужно проверить ошибки
Проверь:
1. Все файлы из `TRACKS` и `MASKS` в `inner_bot.py` реально существуют в `tracks/` и `masks/`
2. JSON-треки валидны (нет синтаксических ошибок, есть поле `question` в каждом шаге)
3. В `.env.example` перечислены все переменные, которые читает `config.py`
4. В `.gitignore` есть `.env`
5. Нет захардкоженных токенов или ключей в Python-файлах

Сообщи о найденных проблемах и предложи исправления.

Выполни по порядку:
1. Проверь ошибки (см. выше)

Не выполняй push самостоятельно — только предложи команды.
